如何自定义WordPress的固定链接

WordPress的后台可以自己设置一些常见的固定链接类型，例如像下图，我们要知道一个好的固定链接也是对SEO优化有着很重要的作用，所以一般来说，固定链接尽量要短，而且尽量避免使用时间类型的，因为你使用时间类型的就相当于告诉蜘蛛爬虫你这篇文章已经过时，蜘蛛爬虫已经不喜欢了，蜘蛛爬虫喜欢新鲜的味道。

那么我们就需要一个好的固定链接了，在这里我推荐使用  【http://域名/分类目录/文章ID.html】的格式，或者更简单的【http://域名/文章ID.html】这种格式。下面我们主要来说一下【http://域名/分类目录/文章ID.html】这种格式，因为这种更适合于归档查看。

首先，打开后台设置页面，找到设置-固定链接-自定义结构，下面的选项可以任意搭配，在这里我稍微讲一下。

%year%    年

%monthnum%    月

%day%    日

%hour%    时

%minute%    分

%second%    秒

%post_id%    文章ID

%postname%    文章名字

%category%    目录

%author%    作者

注：这些都是可以任意搭配使用的

然后我们需要自定义的目录+文章ID.html的形式的话就需要在自定义结构下面填写/%category%/%post_id%.html这样一个格式，如下图：

然后点击保存，然后在发布文章的时候选择自己的分类目录，或者新建一个分类目录，然后发布就成功显示为https://v2vv.cn/love/1.html的形式了。

【注意】：如果打开之后无法显示的话说明服务器空间没有开启HTML静态化，如果用的是宝塔面板的话，可以直接在网站-设置-伪静态里面开启。或者新建一个【.htaccess】文件，里面写入：

location / {
index index.html index.php;
if (-f $request_filename/index.html){
rewrite (.*) $1/index.html break;
}
if (-f $request_filename/index.php){
rewrite (.*) $1/index.php;
}
if (!-f $request_filename){
rewrite (.*) /index.php;
}
}

rewrite /wp-admin$ $scheme://$host$uri/ permanent;

然后保存，上传到根目录即可。
